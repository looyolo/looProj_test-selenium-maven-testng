
[1] 什么是 Behavior Driven 行为驱动？

BDD 鼓励 软件项目中的 开发人员、QA(质量保证人员)、非技术人员（需求分析、数据分析师）之间协作，
通过使用 标准化的自然语言 将 客户、需求人员、开发人员和测试人员 关联在一起协作，
将用户故事(User Story ，用户对产品的一切行为 编织成为一个个故事)
建立起 一一对应 的 需求和用例 之间的 映射关系，
保证 开发和测试的 目标与范围 严格地和 需求 保持一致，
避免 需求理解不一致问题，最终测试结果 也很容易被 参与者各方 理解和认可。

【2】Cucumber 是什么？

Cucumber 是 实现 BDD 的一种主流的测试框架。
它使用 Gherkin 语言来描述 测试功能、测试场景、测试步骤 和 测试结果。
Gherkin 语言 支持 超过 40 种自然语言，包括 中文、英文。
